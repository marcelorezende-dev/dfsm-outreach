Components

Hero / page banner

The light page banner that sits directly under the top bar. It replaced the old dark-green gradient hero: a centered name on the cream paper, with no descriptive body copy. Two variants — the full-name wordmark on Home, and the name + DSL-IP lockup on interior pages such as the Library.

Variant A — full-name wordmark (Home)

Dryland Forest Support Accelerator.

The name is spelled out in full and carries the banner alone — no tagline, no logo. The line break is explicit (Dryland Forest / Support Accelerator.) so the name always sets on exactly two lines rather than rewrapping with the viewport. Wordmark steps down to clamp(28px,5.4vw,68px). Used on DFSA.html.

Variant B — page name + tagline (interior)

Library.

The knowledge library

A short page name at the full wordmark scale, with a one-line serif-italic tagline describing the page. No DSL-IP logo — the programme lockup lives in the footer, not the hero. Used on Library.html.

Shared foundations

Both variants: background = --cream with faint clay + green radial tints (identical recipe to drylands.html). Wordmark = --font-display, uppercase, in --forest-deep; the trailing dot = --amber. Tagline (variant B only) = serif italic in --clay-deep.

Rules

Do

  • Use variant A on Home, where the full name is the whole message.
  • Use variant B on interior pages: short page name + a one-line tagline.
  • Keep the amber trailing dot on the wordmark in both variants.
  • Keep it centered, light, and free of long body copy.

Don't

  • Reintroduce the dark-green gradient / globe / grain background.
  • Put the DSL-IP logo in the hero — it belongs in the footer.
  • Add a descriptive intro paragraph back into the hero.
  • Set the full name at the short-wordmark scale — it will overflow.
  • Repeat the name as a tagline underneath when variant A already spells it out.
