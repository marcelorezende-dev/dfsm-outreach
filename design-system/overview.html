DFSA Design System

Read me first

This is the living design system for the Dryland Forest Support Accelerator platform — the tokens, type, components and page inventory that keep ~35 pages coherent. It is a temporary identity adopted from DSL-IP while the DFSA design language is developed.

How it's organised

Every card in the Design-System pane starts with a first-line <!-- @dsCard group="…" --> marker. Four families:

Foundations Colour · type · spacing, radii & elevation
Components Top bar · buttons · cards · hero · language switcher
Brand Wordmark · DSL-IP / FAO / GEF logos
Pages Every live page, grouped by area

Single source of truth

All tokens and the three type families live in one file.

DFSA/colors_and_type.css defines every colour, the type families & scale, and the radius / shadow / spacing steps as CSS custom properties. Always reference a token (var(--forest), var(--r-md)) rather than a raw value, so a future rebrand propagates from one place. Component cards link this file directly, so they always show the real, current values.

Working rules

Do

  • Compose from tokens and the documented components.
  • Keep the top bar identical on every page (see the Top bar card).
  • Mirror English copy edits into drip-i18n.js rows.

Don't

  • Introduce off-palette colours or one-off type sizes.
  • Fork component markup per page.
  • Treat this identity as final — it is interim (see Brand).
